# OpsBoard Tomcat-Only 운영 체크리스트

사용자 약 10명 규모의 사내 내부 시스템 기준으로, `Tomcat 단독` 운영 시 최소 체크리스트입니다.

## 1) 배포 기본
- `target/opsboard.war` 생성 후 내부 톰캣 `webapps`에 배포
- 컨텍스트 경로는 `/opsboard` 고정
- 톰캣 서비스 재기동 후 정상 접속 확인

## 2) 네트워크/접근 통제 (최소)
- 톰캣 포트(예: 8080)는 내부망에서만 접근 허용
- 서버 방화벽에서 외부 대역 차단
- `manager`, `host-manager` 앱은 비활성화 또는 접근 제한

## 3) JVM 최소 설정
- 운영 서버 메모리에 맞춰 초기/최대 힙 지정
  - 예: `-Xms512m -Xmx1024m`
- 서버 재기동 시 동일 옵션 적용되도록 서비스 설정 고정

## 4) 로그 관리
- `catalina.out`와 애플리케이션 로그 파일 용량 주기 점검
- 로그 롤링(일/용량 기준) 적용
- 장애 시점 추적을 위해 서버 시간대/시간 동기화 유지

## 5) 배포 후 점검 항목
- `/opsboard/home` 접속 확인
- 서버/사이트/계정/이슈 각 메뉴 진입 확인
- DB 연결 및 기본 조회 동작 확인

## 6) 나중에 프록시(Nginx/Apache) 도입하는 시점
- SSL 인증서 운영이 필요할 때
- SSO/엣지 인증이 필요할 때
- 여러 앱 라우팅/무중단 배포 요구가 생길 때
